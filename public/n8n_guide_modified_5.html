選擇教學模組

Codex 教學

Codex 是 OpenAI 的 AI 程式開發助手。你可以把它想成一個會幫你「看懂程式、寫程式、改程式、跑測試、解釋錯誤」的 AI 助理。它不只是聊天回答問題，Codex 可以在專案中閱讀檔案、修改程式、執行指令。

🎯 Codex 適合誰？

1. 完全新手
想學程式，不知道怎麼開始。
2. 初學者
已學點 Python/JS，想加快。
3. 非工程背景
做小工具、網頁或腳本。
4. 開發者
幫忙除錯、重構、補文件。

🔰 為什麼一定要匯入專案？

很多新手打開 Codex 會遇到要求匯入專案且不能跳過。因為 Codex 會直接操作檔案，它必須知道工作目錄在哪。最簡單的解法是在桌面建立一個空資料夾並選取，接著要 Codex 幫你從零開始建立。

我是一個新手，請幫我從零開始建立簡單專案，並教我每個檔案做什麼。

💻 在哪裡用？

強烈推薦：先用 IDE 或 App，再碰 CLI。

  • A. 在 IDE 中用 (推薦): 支援 VS Code、Cursor 等。
  • B. 用 Codex app (推薦): 官方應用程式。
  • C. CLI: 需要熟悉終端機操作。

✨ 可以拿來做什麼？

  • 生出程式草稿
  • 看懂別人的程式
  • 修 bug
  • 整理與優化程式
  • 完成一整段開發流程

🚀 新手上手 5 步驟

  1. 準備小專案: 從小任務起手。
  2. 需求講清楚: 講明語言、功能與風格。
  3. 先看做什麼: 不急著收，看改了什麼檔案。
  4. 要求它解釋: 追問為什麼這樣寫，讓它變成教練。
  5. 小步修改: 不要一次改太多功能。

什麼是 Claude？

Claude 是由 Anthropic 公司開發的 AI 助理，你可以把它想像成一位隨時待命、知識淵博的工作夥伴。它能用自然的對話方式理解你的需求，幫你處理各種文字工作、回答問題、提供建議。

與搜尋引擎不同，Claude 不只是找資料，而是理解你的問題、思考分析後給出整合性的回答。

💬

自然對話

用日常語言溝通，不需要特殊指令或程式語言

🧠

深度理解

理解上下文與複雜指令，不只是關鍵字搜尋

✍️

文字創作

撰寫、修改、翻譯、摘要各類文件

🔍

分析推理

協助整理想法、分析問題、提供多角度建議

💡
重要認知：Claude 的知識有截止日期（目前約到 2025 年中），對於最新的新聞、即時資料，建議搭配其他工具確認。

開始使用：第一次登入

  1. 前往官網：開啟瀏覽器，前往 claude.ai，點擊「Sign Up」註冊，或「Log In」登入既有帳號。
  2. 建立帳號：用 Google 帳號快速登入，或輸入 email 建立新帳號。免費版即可開始使用，無需信用卡。
  3. 進入主頁面：登入後會看到對話框，下方有輸入欄位，直接打字即可開始對話。
  4. 發送第一個訊息：試著輸入「你好，請介紹一下你自己的能力」，按 Enter 送出，看看 Claude 怎麼回應！
📌
免費版限制：每日有訊息使用上限，適合日常輕量使用。若需要更多使用量，可升級至 Claude Pro（付費版）。

如何問出好答案：Prompt 基礎

Prompt 就是你輸入給 Claude 的指令或問題。問的方式不同，得到的答案品質差異很大。

🎯
好 Prompt 的公式：
背景 / 角色（我是誰、什麼情境） ＋ 具體任務（要做什麼） ＋ 格式要求（怎麼呈現） ＋ 限制條件（長度、語氣、對象）

範例比較

寫法範例
❌ 模糊幫我寫一封信
✅ 清楚幫我寫一封感謝信給客戶陳先生，感謝他長期支持，語氣要正式但親切，約 150 字以內
❌ 抽象解釋 AI
✅ 有對象用小學生能理解的方式解釋 AI 是什麼，請用生活中的比喻，不要用專業術語

5 個立即可用的萬用技巧

  1. 指定角色：你是一位資深的行銷專家，請幫我分析這個產品的目標市場…
  2. 要求條列格式：請用條列式整理以下會議記錄的重點，每點不超過 20 字…
  3. 指定字數：請用 3 句話總結以下文章的核心觀點…
  4. 要求多個選項：請提供 3 種不同風格的標題，分別是正式、活潑、疑問句風格…
  5. 追問與修改：剛才的第 2 點說明不夠清楚，請用更具體的例子重新解釋…

新手常見問題 Q&A

Q：我的對話內容會被保留嗎？
對話記錄在同一個對話視窗中會保留，讓 Claude 記得你說過的內容。但開新對話後，Claude 不會記得上次說了什麼，每個對話是獨立的。
Q：Claude 記得我說的話到什麼程度？
在單一對話中，Claude 能理解數千字的對話歷史。重要的背景資訊建議每次都簡短提醒。
Q：我輸入的資料安全嗎？
避免輸入極度敏感的個人資訊或公司機密。Anthropic 有隱私政策，但使用 AI 工具時保持基本的資訊安全意識是好習慣。
Q：Claude 給的答案不滿意怎麼辦？
直接告訴它！你可以說「剛才的回答太複雜了，請用更簡單的方式重寫」或「請把重點放在第 3 點並展開說明」。Claude 很善於根據回饋調整。
Q：Claude 和 ChatGPT 有什麼不同？
兩者都是大型語言模型 AI，功能相近。Claude 在長文本處理、遵循指令方面有自己的特點。建議兩者都嘗試，找到最適合你工作流程的工具。

Projects：建立你的 AI 工作空間

一般對話是一次性的——關掉視窗就結束了。Projects 讓你建立一個持久的工作空間，裡面放參考文件和固定指令，同一專案下所有對話都共享這些資料。

設定四步驟

  1. 建立專案：在左側欄點「+ New Project」，以用途命名：「行銷企劃」「客服中心」「季報撰寫」
  2. 上傳參考文件：支援 PDF、Word、試算表、程式碼。可放：產品手冊、品牌語氣指南、常見問題集。
  3. 設定專案指令（Custom Instructions）：定義 Claude 的角色、語氣、輸出格式，以及需要記住的規則。
  4. 開始對話：專案內每次對話，Claude 都會自動讀取文件和指令，不需重複說明。

專案指令範本

📋
客服回覆專案：你是「○○品牌」的客服助理。請根據上傳的產品手冊和 FAQ 回答問題。語氣友善簡潔，每則回覆不超過 150 字。手冊中找不到答案時，請回覆「感謝您的詢問，我將為您轉接專人處理」，不要自行猜測。
📋
內容行銷專案：你是我們的內容行銷助理。請參考上傳的品牌語氣指南撰寫所有內容。目標受眾：25–40 歲台灣科技業上班族。語氣：專業但不生硬。全部用繁體中文，標題慣用疑問句或數字型（如「5 個方法…」）。

Memory：Claude 記住你這個人

Memory 讓 Claude 在對話結束後，仍然記住關於你的重要資訊。下次開新對話，Claude 會主動讀取這些記憶，像一位真正認識你的助理。

記憶方式說明
🤖 自動擷取Claude 在對話中自動判斷哪些資訊值得記住，主動儲存，不需要你特別說。
✍️ 手動儲存在對話中說「請記住我偏好用條列格式」，Claude 立即存入記憶庫，下次自動套用。

值得讓 Claude 記住的事

  • 角色與背景：「我是○○公司的行銷總監，主要負責 B2B 客戶，對象多為製造業。」
  • 格式偏好：「我喜歡條列式輸出，不要太多鋪陳」「給建議時請直接講重點」
  • 禁止事項：「不要用中國用語，請用台灣慣用繁體中文」
  • 常用任務格式：「我常需要把英文會議摘要翻成正式繁中，請記住這個任務的輸出格式。」
⚙️
到 claude.ai → Settings → Memory，可以查看、編輯、刪除所有已儲存的記憶，完全透明可控。

Skills：把你的 SOP 教給 Claude

Skills 讓你把完整的工作流程「教」給 Claude：觸發條件、輸入格式、執行步驟、輸出格式全部定義清楚。日後只需呼叫這個 Skill，Claude 就按你的 SOP 執行，輸出永遠一致。

💡
Skills 解決什麼痛點？你花 10 分鐘寫出一個完美的 Prompt，之後卻記不住怎麼寫？Skills 就是把這個 Prompt 永久儲存並標準化，下次一鍵呼叫，每次輸出都一樣好。

建立路徑

Claude Desktop → Settings → Customize → Skills → + 建立新 Skill

四個核心設定：名稱（做什麼）、描述（何時呼叫）、內容（如何執行）、範例（輸入/輸出示例）。

三個可直接套用的範本

Skill 1：週報自動整理器

觸發時機：使用者提供本週工作記錄（流水帳、口述或條列皆可）

輸出格式：

📊 本週工作週報 [日期範圍]
✅ 本週完成事項（要有具體描述）
🔄 進行中的項目（附預計完成時間）
📅 下週計畫（附優先順序：🔴高 / 🟡中 / 🟢低）
🙋 需要支援 / 待確認事項
💡 本週亮點一句話
Skill 2：競品快速分析

觸發時機：使用者提供競品名稱

分析維度（7 點）：定位與目標市場、產品功能與特色、定價策略、通路與行銷方式、品牌與內容風格、已知優劣勢、對我方的威脅與機會

輸出：Markdown 表格 + 100 字以內總結，不確定資訊標注（需查證）

Skill 3：客戶來信快速處理

觸發時機：使用者貼上客戶來信

步驟一：分析來信 → 緊急程度（🔴高 / 🟡中 / 🟢低）、問題類型、核心訴求一句話

步驟二：草擬回覆（繁中，正式但不生硬）

特別注意：遇到投訴，語氣要同理，先致歉再解釋

Cowork：讓 Claude 真正替你完成任務

一般 Claude 對話Claude Cowork
Claude 告訴你「怎麼做」，你自己去執行Claude 直接幫你做完，你只需看結果
💻
需要 Claude Desktop：前往 claude.ai/download 下載桌面版（macOS / Windows），登入後在介面頂部切換到 Cowork 模式。

四大核心能力

  1. 直接操作本機檔案：讀寫你授權的本機資料夾，不需手動上傳下載。
  2. 產出可直接使用的文件：含公式的 Excel、PowerPoint 簡報、格式化 Word，可繼續用 Claude 編輯。
  3. 排程任務（定期自動執行）：設定一次，Claude 在指定時間自動執行。
  4. 多步驟研究與分析：搜尋網路、整合資料、生成報告，一個指令完成。

實際範例

  • 「幫我整理 Downloads 資料夾，按類型分類，整理前先給我看計畫，確認後再執行。」
  • 「每週五 17:00，從 Notion 工作記錄抓本週完成任務，整理成週報，傳到 Slack #週報 頻道。」
  • 「搜尋台灣 SaaS 市場最新報告，整理出前 5 大競品定價與功能差異，存到 Research 資料夾。」
⚠️
Cowork 會對你的電腦執行真實動作。在授權任何不可逆的操作（刪除、對外發送、發布）前，請仔細確認計畫內容。執行前 Claude 會先展示計畫讓你確認。

OpenClaw 是什麼？

OpenClaw 是一個可以自己架設的個人 AI 助手平台。你把它安裝在自己的電腦上， 它會在背景啟動一個稱為 Gateway 的服務。

Gateway 負責三件事：

  • 🤖 連接 AI 模型（Claude、GPT-4、Gemini 等）
  • 💬 處理各種通訊管道（Telegram、WhatsApp、Discord 等）
  • 🛠 執行工具（搜尋網頁、讀寫檔案、執行指令等）

你的資料全部留在自己的電腦上，不需要透過任何第三方雲端服務。

2

安裝前準備

① Node.js

OpenClaw 需要 Node.js。建議安裝 Node 24，最低支援 Node 22.14+。

打開終端機（Terminal），輸入以下指令確認你是否已經安裝：

node --version

如果顯示版本號（例如 v24.0.0），代表已安裝。 如果出現錯誤，請先到 nodejs.org 下載安裝。

💡
不確定什麼是終端機？
macOS：按 Command＋空白鍵，搜尋「Terminal」，按 Enter 開啟。
Windows：建議安裝 WSL2，再用 WSL 終端機操作，體驗更穩定。
Terminal 
$ node --version
v22.22.2

② AI 模型的 API Key

你需要至少一個 AI 模型供應商的帳號與 API Key。API Key 像是一組密碼， 讓 OpenClaw 可以使用那個模型的服務（通常依用量計費）。

任選一個申請即可：

  • Anthropic（Claude）：console.anthropic.com
  • OpenAI（GPT）：platform.openai.com
  • Google（Gemini）：aistudio.google.com

Onboarding 過程中會請你輸入 API Key，先複製好備用。

✅
準備好了嗎？確認清單：
  • Node.js 已安裝（node --version 有輸出版本號）
  • 已取得至少一個 AI 供應商的 API Key

安裝 OpenClaw

安裝腳本會自動偵測你的環境，並完成所有必要的安裝步驟。過程大約 1～2 分鐘。

確認安裝成功

安裝完成後，執行以下指令確認 CLI 可以使用：

openclaw --version

看到版本號（例如 openclaw/1.x.x）就代表成功了。

⚠️
看到「openclaw: command not found」？

執行 Onboarding 向導

Onboarding 向導會一步一步引導你完成所有基本設定。在終端機執行：

💡
--install-daemon 是什麼？
加上這個參數，Gateway 服務會在你的電腦開機時自動啟動， 你不需要每次手動開啟它。建議第一次安裝都加上。

向導步驟說明

向導會依序問你幾個問題，以下是每個步驟的說明：

  1. QuickStart 還是 Advanced？
    選 QuickStart。它使用預設值，適合大部分情況。 Advanced 是給有特殊需求的進階用戶。
  2. 選擇 AI 模型供應商 + 輸入 API Key
    用方向鍵選擇你的供應商（例如 Anthropic），然後輸入你的 API Key。 輸入時畫面不會顯示字元（這是正常的保護機制）。
  3. Workspace 路徑
    這是 AI 助手存放檔案的工作目錄。預設是 ~/.openclaw/workspace， 直接按 Enter 接受即可。
  4. Gateway 設定
    預設 port 是 18789。除非你有特殊需求，直接按 Enter 略過所有選項。
  5. 通訊管道設定（可略過）
    這裡可以設定 Telegram、WhatsApp 等通訊管道。 第一次可以直接跳過，等基本設定完成後再回來設定。
  6. 安裝 Daemon
    確認要安裝背景服務，按下 Enter 或選「Yes」。
  7. 健康檢查
    向導會自動啟動 Gateway 並確認一切正常。

整個過程大約 2 分鐘。

通訊管道設定

OpenClaw 支援多種通訊管道，最常用的是 Telegram。 管道設定都儲存在 openclaw.json 的 channels 區段。

📱 Telegram

Telegram 需要一個 Bot Token。如果還沒建立 Bot，先到 @BotFather 執行 /newbot，取得 Token 後再繼續。

設定 Bot Token：

Terminal 
$ openclaw config set channels.telegram.botToken YOUR_TOKEN_HERE
✓ channels.telegram.botToken updated
$ openclaw gateway restart
✓ Gateway restarted

建議用環境變數存放 Token，避免明文寫入 config 檔：

Terminal 
$ openclaw config set channels.telegram.botToken \
    --ref-provider default --ref-source env --ref-id TELEGRAM_BOT_TOKEN
# 把 TELEGRAM_BOT_TOKEN=xxx 加進 ~/.zshrc 或 ~/.bashrc

群組策略（groupPolicy）

控制 Bot 在群組中的行為：

值說明
open允許所有群組訊息（不建議，可能被陌生人觸發）
allowlist只允許 groupAllowFrom 清單中的群組/用戶（預設值，最安全）
deny完全拒絕群組訊息
⚠️
設了 allowlist 但沒加任何 ID？
所有群組訊息會被靜默丟棄。用 openclaw directory self 查自己的 ID， 或 openclaw doctor 會提示你這個問題。

加入允許的群組 / 用戶 ID：

openclaw config set channels.telegram.groupAllowFrom '["123456789","group_id"]' --strict-json

dmPolicy（私訊策略）

值說明
open任何人都可以私訊 Bot
pairing需要先完成配對才能使用（預設）
deny拒絕所有私訊

其他管道需透過互動式設定向導來設定（Token/Cookie 等驗證流程較複雜）：

openclaw configure

或針對特定管道：

openclaw channels login --channel discord
openclaw channels login --channel whatsapp

查詢已連線管道的狀態： 
openclaw status

安全與存取控制

🔒 Exec 工具安全模式（Sandbox）

控制 AI 是否可以執行 shell 指令，分為三個層級：

模式說明
deny完全禁止執行任何 shell 指令（最安全）
allowlist只允許白名單內的指令（需手動審批新指令）
full允許執行所有指令（開發者模式，需謹慎）
 
openclaw config set agents.defaults.exec.security allowlist
🚨
full 模式有風險
設為 full 時，AI 可以執行任意 shell 指令，包含刪除檔案、發送網路請求等。 只在你完全信任 AI 行為，且了解風險時使用。

審批機制（Approvals）

在 allowlist 模式下，遇到未知指令時，AI 會暫停並等待你的審批：

Terminal 
# 查看待審批的指令
$ openclaw approvals list

# 單次允許
$ openclaw approvals allow-once <id>

# 永久加入白名單
$ openclaw approvals allow-always <id>

Gateway Bind 設定

控制 Gateway 對外的可見範圍：

設定說明
loopback只允許本機連線（127.0.0.1），最安全（預設）
local允許區域網路連線
all對所有介面開放（需搭配防火牆）
 
openclaw config set gateway.bind loopback

API Key 安全存放

避免 API Key 明文寫入設定檔，改用環境變數參照：

# ~/.zshrc 或 ~/.bashrc 加入：
export ANTHROPIC_API_KEY="sk-ant-..."
export TELEGRAM_BOT_TOKEN="bot..."

# 在 config 中參照環境變數
openclaw config set secrets.providers.default \
  --provider-source env \
  --provider-mode flat
# 建立 JSON 格式的 secret 檔案
{
  "ANTHROPIC_API_KEY": "sk-ant-...",
  "TELEGRAM_BOT_TOKEN": "bot..."
}

# 在 config 中參照
openclaw config set secrets.providers.vault \
  --provider-source file \
  --provider-path /etc/openclaw/secrets.json \
  --provider-mode json

安全稽核

openclaw security audit --deep

Workspace 設定

Workspace 是 AI 助手的「工作目錄」，預設位於 ~/.openclaw/workspace。 AI 讀寫的所有檔案都在這裡，也是記憶體系統的根目錄。

核心檔案角色

檔案說明
SOUL.md定義 AI 的個性、風格與核心原則（每次啟動都會讀取）
USER.md你的基本資訊：姓名、時區、偏好語言、備注
MEMORY.md長期記憶，AI 在主要會話中讀寫的持久記憶
AGENTS.mdAI 的行為規則與工作指引（類似系統 Prompt 的補充）
TOOLS.md你的環境筆記（SSH、攝影機、TTS 偏好等）
HEARTBEAT.md定義心跳輪詢時的檢查任務清單
memory/YYYY-MM-DD.md每日原始日誌，AI 用來記錄當天事件
✅
客製化 AI 行為的最快方式：
直接編輯 SOUL.md 或 AGENTS.md，下次對話就會生效，不需要重啟 Gateway。

Heartbeat 與排程

Heartbeat（心跳輪詢）

Gateway 每隔一段時間（預設 30 分鐘）會自動喚醒 AI，讓它執行 HEARTBEAT.md 中定義的任務。 這是讓 AI 主動觀察、主動通知你的機制。

HEARTBEAT.md 範例：

# 每次心跳輪詢時的任務

## 要做的事
- 檢查有沒有未讀重要郵件
- 確認接下來 2 小時內有沒有行事曆事項
- 如果距離上次報告超過 8 小時，主動打招呼

## 不用做的事
- 23:00~08:00 不用主動推送（除非緊急）
- 沒有新資訊時靜默即可

調整 Heartbeat 間隔

openclaw config set agents.defaults.heartbeat.intervalMinutes 60

Cron 排程任務

需要精確時間或獨立執行的任務，用 Cron：

Terminal 
# 列出所有 Cron 任務
$ openclaw cron list

# 新增 Cron 任務（每天早上 9 點傳早安摘要到 Telegram）
$ openclaw cron add \
    --schedule "0 9 * * *" \
    --message "產生今日天氣與行事曆摘要" \
    --deliver --target telegram:8473859266

# 移除 Cron 任務
$ openclaw cron remove <id>

Heartbeat vs Cron：選哪個？

情境建議
多項定期檢查（郵件 + 行事曆 + 天氣）Heartbeat
精確時間很重要（每天 9:00 整）Cron
一次性提醒（20 分鐘後叫我）Cron
需要用到最近對話的上下文Heartbeat
獨立執行、不需要主 session 參與Cron

遠端連線

預設 Gateway 只開放本機連線（loopback），如果你想從手機 App 或外部網路連入，需要額外設定。

Tailscale（推薦方案）

Tailscale 是最安全的遠端連線方式，不需要開放公網 Port， 所有流量走加密 VPN Mesh。 

# 先安裝 Tailscale：https://tailscale.com/download
# 登入後確認裝置已在網路中

# 開啟 Tailscale 整合
openclaw config set gateway.tailscale.mode auto
openclaw gateway restart
💡
手機 App 連線：
設定好 Tailscale 後，執行 openclaw qr 產生配對 QR Code， 用 iOS/Android 的 OpenClaw App 掃描即可。

Reverse Proxy（進階）

如果部署在 VPS 或需要自訂 domain，可以用 Nginx/Caddy 當反向代理：

# Nginx 範例（/etc/nginx/conf.d/openclaw.conf）
server {
    listen 443 ssl;
    server_name your.domain.com;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}

同時把 Gateway bind 改為 loopback（只讓 Nginx 連）：

openclaw config set gateway.bind loopback
openclaw config set gateway.controlUi.publicUrl https://your.domain.com
openclaw gateway restart
⚠️
對外開放時務必啟用 HTTPS
Gateway 的 auth token 在 HTTP 下以明文傳輸。 建議使用 Caddy（自動管理 Let's Encrypt 憑證）或 Tailscale HTTPS。

DNS 探索（多機環境）

在多台機器組成的環境中，可以用 DNS 讓各裝置自動找到 Gateway：

openclaw dns --help

常見問題

改了設定後，什麼時候要重啟 Gateway？

大部分設定改完都需要重啟才會生效：

openclaw gateway restart

例外：直接編輯 SOUL.md、AGENTS.md、HEARTBEAT.md 等 workspace 檔案，下次對話就會生效，不需要重啟。

如何確認 config 目前的值？ 
# 查看特定路徑的值
openclaw config get gateway
openclaw config get agents.defaults.model
openclaw config get channels.telegram

# 查看設定檔位置
openclaw config file
要怎麼重置 / 清除設定？

移除特定設定（恢復預設值）：

openclaw config unset agents.defaults.model.primary

完整重置（保留 CLI，清除所有設定和資料）：

openclaw reset
記憶搜尋（Memory Search）要怎麼設定？

記憶搜尋需要 embedding 模型的 API Key（OpenAI、Google、Voyage、Mistral 其一），或本地 embedding 模型。

# 設定方式
openclaw configure --section model

# 停用（如果不需要語意搜尋）
openclaw config set agents.defaults.memorySearch.enabled false

# 確認狀態
openclaw memory status --deep
Skills 自訂安裝路徑怎麼設？

可以把自訂 Skill 放在 workspace 底下或任意路徑，然後在 config 指定：

openclaw config set agents.defaults.skills.extraPaths \
  '["/path/to/my-skills"]' --strict-json
多個 Telegram Bot 可以同時運作嗎？

一個 Gateway 實例目前只支援一個 Telegram Bot。 如果需要多 Bot，可以用 --profile 參數開多個獨立 Gateway：

openclaw --profile work gateway start
openclaw --profile personal gateway start

各 profile 使用不同的 state 目錄和設定檔，互不干擾。
